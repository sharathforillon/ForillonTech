The page must let visitors pick features, submit contact info, and track feature interest and conversions. Requirements summary.

Visual. Use IBM Plex Sans for all UI. Use a vibrant professional palette. Provide a modern, premium layout with generous white space, subtle card shadows, and micro-animations for interactions.

UX. Feature selection must be frictionless. Show two grouped lists. Selected features are stored and submitted with the contact form. The flow must be: Hero -> Feature Explorer -> Minimal Form modal -> Thank-you with calendar CTA.

Tracking. Track unique visitors, feature selections, CTA clicks, and form submissions. Provide GA4 event names and LinkedIn/Meta pixel snippets.

Performance. Lighthouse score > 90 on mobile and desktop. Keep bundle small. Provide server-side rendering for initial load.

Accessibility. WCAG AA. Keyboard focus states, proper alt text, semantic HTML, ARIA for toggles.

Integration. Save leads to HubSpot or a provided webhook endpoint. Include optional Calendly link on Thank-you page.

Deliverables. Complete Next.js repo, Tailwind config, environment variable sample (.env.example), deployment steps to Vercel or Replit, testing checklist, and a short README.
